What is FAI? Main Features

  • done  A tool for automated unattended installation. Lazy system administrators like it.
  • done  Remote network installation of different Linux flavors
  • done  Easy-to-use centralized management system for your Linux deployment.
  • done  FAI profiles extend the FAI class concept
  • done  FAI shows a curses based menu for selecting a profile
  • done  It's fast. It only takes a few minutes for a complete installation.
  • done  Scalable. FAI users manage their computer infrastructures starting from a few computers up to several thousands of machines.
  • done  Easy creation of customized Linux installation media like CD, DVD or USB stick
  • done  Autodiscover of the FAI install server when doing a network installation
  • done  Different hardware and different configuration requirements are easy to establish using FAI. You do not need to repeat information that is shared among several machines.
  • done  Using the FAI class concept, you can group a bunch of similar machines.
  • done  Installation targets: desktops, servers, notebooks, Beowulf cluster, rendering or web server farm, Linux laboratory or classroom.
  • done  Linux rollout, mass installation and automated server provisioning are additional topics of FAI.
  • done  FAI is lightweight. No special daemons are running, no database setup is needed.
    It's architecture independent, since it consists only of shell, Perl and Cfengine scripts.
  • done  Besides initial installations, it is also used for daily maintenance.
  • done  Creating customized chroot environments, also cross-architecture for e.g. for embedded devices
  • done  Building your own disk image for cloud instances, also cross-architecture
  • done  Cloning machines using disk images is supported
  • done  Also bare-metal provisioning and deploying virtual machines is supported
  • done  Compared to tools like kickstart or cobbler for Red Hat, autoyast for SUSE or Jumpstart for SUN Solaris, FAI is much more flexible.
    You can tune every small part of your configuration to your local needs using hooks.
  • done  More technical information in the flyer and poster

FAI Installation Steps

  • arrow_forward  Network boot via PXE
  • arrow_forward  Receive configuration data via HTTP, NFS, svn or git
  • arrow_forward  Run scripts to determine FAI classes and variables
  • arrow_forward  Partition local hard disks and create RAID, LVM configuration and the file systems
  • arrow_forward  Install and configure software packages
  • arrow_forward  Customize OS and software to your local needs
  • arrow_forward  Reboot freshly installed machine
  • arrow_forward  All this can also be done via an unattended CD installation

FAI History

FAI is rock-solid. It's now more than 16 years old. FAI was started in 1999 at the University of Cologne, when Thomas had to organize an installation of a Linux cluster. Today, FAI is regularly used in varying environments, ranging from a dozen machines up to several thousands machines.
